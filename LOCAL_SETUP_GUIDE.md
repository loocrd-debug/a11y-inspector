## 1. 왜 로컬에서 실행해야 하나요?

이 도구는 **www.gov.kr** (정부24 민원안내) 페이지를 검사합니다.

문제는 Railway 같은 **해외 클라우드 서버**에서는 한국 정부 서버에 접속이 **차단**됩니다.

```
Railway (미국 서버) → www.gov.kr ❌ 접속 차단
내 PC (한국 IP)     → www.gov.kr ✅ 정상 접속
```

**→ 민원 검사를 하려면 반드시 내 PC에서 직접 실행해야 합니다.**

## 2. 사전 준비 — Node.js 설치

Node.js는 이 프로그램을 실행하는 데 필요한 **기본 엔진**입니다.

### Windows

1. 브라우저에서 **https://nodejs.org** 접속
2. **"LTS" 버전** (왼쪽 초록 버튼) 클릭하여 다운로드
   - 예: `node-v22.x.x-x64.msi`
3. 다운로드된 `.msi` 파일 실행
4. 설치 마법사에서 **계속 "Next"** → 마지막에 **"Install"** 클릭
5. 설치 완료 후 **재시작하지 않아도 됩니다**

**설치 확인:**
- 키보드 `윈도우키 + R` → `cmd` 입력 → 확인
- 검은 창(명령 프롬프트)에서 아래 명령 입력:

```
node --version
```

- `v22.x.x` 같은 버전 번호가 나오면 **성공** ✅
- 아무것도 안 나오거나 오류가 나면 설치를 다시 하세요.

### macOS

1. 브라우저에서 **https://nodejs.org** 접속
2. **"LTS"** 버전 클릭 → `.pkg` 파일 다운로드
3. 다운로드된 파일 실행 → "계속" 클릭 → 설치

**설치 확인:**
- `Cmd + Space` → "터미널" 검색 → 터미널 실행
- 아래 명령 입력:

```
node --version
```

## 3. 소스 코드 다운로드

### 방법 A: ZIP으로 다운로드 (Git 몰라도 OK, 추천)

1. 브라우저에서 **https://github.com/loocrd-debug/a11y-inspector** 접속
2. 초록색 **`< > Code`** 버튼 클릭
3. **`Download ZIP`** 클릭
4. 다운로드된 `a11y-inspector-main.zip` 파일을 압축 해제
   - Windows: 파일 우클릭 → "모두 추출"
   - 적당한 위치에 압축 해제 (예: `C:\Users\내이름\a11y-inspector`)
5. 압축 해제된 폴더 이름이 `a11y-inspector-main`이라면 `a11y-inspector`로 바꿔도 됩니다.

### 방법 B: Git으로 클론 (Git이 설치된 경우)

명령 프롬프트(또는 터미널)에서:

```bash
git clone https://github.com/loocrd-debug/a11y-inspector.git
cd a11y-inspector
```

## 4. 의존성 패키지 설치

프로그램 실행에 필요한 라이브러리들을 설치합니다.

### 명령 프롬프트(터미널) 열기

**Windows:**
- `윈도우키 + R` → `cmd` → 확인
- 또는 파일 탐색기에서 압축 해제한 폴더를 열고 → 주소창 클릭 → `cmd` 입력 → Enter

**macOS:**
- 터미널 실행 후 압축 해제한 폴더로 이동

### 폴더로 이동

압축 해제한 폴더 경로로 이동합니다.

**Windows 예시:**
```cmd
cd C:\Users\내이름\a11y-inspector
```

**macOS 예시:**
```bash
cd ~/Downloads/a11y-inspector
```

> 💡 **팁**: 파일 탐색기에서 폴더를 열고, 주소창에 `cmd` 를 입력하면 해당 폴더에서 명령 프롬프트가 열립니다.

### 패키지 설치 실행

```
npm install
```

- 실행하면 수백 개의 패키지가 설치됩니다.
- 인터넷 속도에 따라 **1~5분** 정도 걸립니다.
- 마지막에 `added XXX packages` 메시지가 나오면 **성공** ✅

> ⚠️ 경고(warn) 메시지가 나와도 괜찮습니다. **error**가 없으면 정상입니다.

## 5. 브라우저(Chromium) 설치

이 도구는 **보이지 않는 브라우저(headless Chromium)**를 사용해 웹사이트를 검사합니다.  
아래 명령으로 해당 브라우저를 설치합니다.

```
npx playwright install chromium
```

- 약 **150~300MB** 다운로드됩니다.
- 인터넷 속도에 따라 **2~5분** 소요됩니다.
- 마지막에 `✓ Chromium ...` 메시지가 나오면 **성공** ✅

**Windows에서만 추가로 실행:**

```
npx playwright install-deps chromium
```

> ⚠️ macOS는 위 명령이 필요 없을 수 있습니다. 오류가 나면 무시하세요.

## 6. 서버 실행

이제 모든 준비가 끝났습니다! 서버를 시작합니다.

```
node server.js
```

아래와 같은 메시지가 나오면 **서버가 성공적으로 시작된 것**입니다 ✅:

```
✅ 가디언즈 오브 겔럭시 서버 실행 중: http://0.0.0.0:3000
📋 민원목록 엑셀 로딩 중...
✅ 민원목록 로드 완료: 1차=1963, 2차=3237, 3차=3261
```

### 브라우저에서 접속

서버가 실행된 상태에서 웹 브라우저를 열고:

```
http://localhost:3000
```

→ **"가디언즈 오브 겔럭시"** 메인 화면이 나타납니다 🎉

### 민원 검사 페이지 접속

```
http://localhost:3000/minwon.html
```

## 7. 민원 검사 사용 방법

### 7-1. 전환 차수별 검사 (엑셀 기준)

페이지 중간에 **"전환 차수별 검사 (엑셀 기준)"** 섹션이 있습니다.

![차수별 검사 섹션](섹션 위치: 배치 스캔 탭 → 하단)

| 버튼 | 검사 대상 | 건수 |
|------|-----------|------|
| 🔵 1차 전환 민원 검사 | 1차 전환 대상 민원 | 1,963건 |
| 🟢 2차 전환 민원 검사 | 2차 전환 대상 민원 | 3,237건 |
| 🟣 3차 전환 민원 검사 | 3차 전환 대상 민원 | 3,261건 |

**검사 시작 순서:**

1. `http://localhost:3000/minwon.html` 접속
2. **"배치 스캔 실행"** 탭 클릭
3. 원하는 차수 버튼 클릭 (예: "1차 전환 민원 검사")
4. 확인 팝업에서 **"확인"** 클릭
5. 진행 상황이 화면에 표시됩니다
6. 검사 완료 후 **"검사 결과 & 증적"** 탭에서 결과 확인

> ⚠️ **주의**: 1,963건을 모두 검사하면 **수 시간** 소요됩니다.  
> 일부만 테스트하려면 "민원 목록 조회" 탭에서 몇 개 선택 후 "배치에 추가" → "배치 스캔" 방식을 사용하세요.

### 7-2. 서버를 종료할 때

명령 프롬프트에서 `Ctrl + C` 를 누르면 서버가 종료됩니다.

### 7-3. 다음에 다시 실행할 때

폴더에서 명령 프롬프트를 열고 아래 명령만 입력하면 됩니다:

```
node server.js
```

(npm install, playwright install은 **한 번만** 하면 됩니다.)

## 8. 자주 묻는 질문 / 오류 해결

### ❓ `node` 명령을 찾을 수 없습니다 / `'node'은(는) 내부 또는 외부 명령 아닙니다`

**원인**: Node.js가 제대로 설치되지 않았거나 PC를 재시작해야 합니다.

**해결**:
1. PC를 재시작한 후 다시 시도
2. 그래도 안 되면 Node.js를 다시 설치

---

### ❓ `npm install` 중 오류 발생

**해결**:
1. 인터넷 연결 확인
2. 회사 네트워크라면 방화벽/프록시 설정 확인
3. 아래 명령으로 캐시 초기화 후 재시도:
```
npm cache clean --force
npm install
```

---

### ❓ `npx playwright install chromium` 중 오류

**해결 (Windows)**:
```
npm install @playwright/browser-chromium
npx playwright install chromium
```

---

### ❓ 서버 실행 후 `http://localhost:3000` 접속이 안 됩니다

**원인**: 포트 3000이 이미 사용 중입니다.

**해결**:
- 명령 프롬프트에서 `Ctrl + C`로 기존 서버 종료 후 다시 실행
- 또는 다른 포트로 실행:
```
PORT=3001 node server.js
```
→ `http://localhost:3001` 로 접속

---

### ❓ 검사 중 오류가 자꾸 납니다

**원인**: 메모리 부족 또는 Chromium 실행 오류

**해결**:
```
node --max-old-space-size=2048 server.js
```

---

### ❓ Windows에서 Chromium 실행 오류

```
npx playwright install-deps chromium
```

또는 Visual C++ 재배포 패키지 설치:  
https://learn.microsoft.com/ko-kr/cpp/windows/latest-supported-vc-redist

---

### ❓ 엑셀 파일을 찾을 수 없다는 오류

`data/minwon-list.xlsx` 파일이 있는지 확인하세요.

없다면 GitHub에서 ZIP으로 다시 다운로드하거나, 아래 경로에 파일이 있는지 확인:
```
a11y-inspector/
  └── data/
        └── minwon-list.xlsx  ← 이 파일이 있어야 합니다
```
